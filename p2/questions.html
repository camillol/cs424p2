Questions


    Group 1

    How did you handle cleaning up the data with multiple instances of same people?

    In the transcripts on the wiki, the first time a name appears in an episode it is linked to the page about that character. We took this data and generated a list of URLs associated with multiple names, and names associated with multiple URLs. You can find this data in the file name_url.txt in our data folder.

    Since some characters share the same page on the wiki (e.g., Fry's parents), we did not blindly unify characters that share the same URL, but we went through the list and checked the mappings manually. We also detected case variations and unified those.

    We chose not to unify variants of characters from other dimensions and the like (e.g. "Bender duplicate #2" or "Amy 420").

    Group 6

    What animation library do you use? How to create the transicion? (sic)

    There is no library. The animation effects are based on two classes I wrote:

    • The Animator class gradually changes a float from its current value to a specified target. It is based on Fry's (no, the other Fry) Integrator class, with several changes (automatic registration, automatic animation stop). The physical model is a spring following Hooke's law, with the object experiencing linear drag, although this may not be obvious from the code.
    • The View class provides a simple but effective view system which allows each part of the interface to work in a local coordinate system. This decouples its code from the position of the view in the superview, making it easy to move the view around together with all of its contents.

    Group 8

    How does the user know which characters & variables are represented in the pie chart?
    Colors for the characters were chosen based on the appearance (dominating color) of the character. The user can identify the color of the character based on the highlight over the button of that particular character. For other characters (characters) we alternated between white and black (for pie chart) to show their presence.
